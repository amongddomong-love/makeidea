카페24 vs Netlify 한 번에 이해하기

“왜 카페24에서 주소를 만들고, 다시 Netlify에 등록했을까?”
도메인(주소)와 호스팅(집)의 역할을 한 장으로 정리한 실무용 설명입니다.

초보도 5분 안에 이해 가능 · 대상: 동료, 친구, 후배에게 설명할 때

한눈에 요약 TL;DR

웹사이트를 만들 때 보통 두 가지를 분리해서 생각합니다.
① 사람에게 보여줄 예쁜 주소(도메인) ② 실제 파일이 올라가는 집(호스팅).

  • 카페24 → 주소(도메인) 담당 · “길 안내판” 역할
  • Netlify → 호스팅 담당 · “집(웹사이트 파일이 있는 곳)” 역할
  • 둘을 연결하면: 예쁜 주소 + 내가 만든 사이트 조합 완성
이전에 했던 작업
1) 카페24에서 도메인 생성

my-site.com 같이 외우기 좋은 이름을 만들고,
“이 주소를 어느 서버로 보낼지” DNS 설정을 준비했습니다.

이전에 했던 작업
2) Netlify에 사이트 배포

index.html이 들어 있는 폴더를 Netlify에 올려서,
실제로 웹사이트가 돌아가는 “집”을 만든 뒤 도메인을 연결했습니다.

i 도메인을 사지 않아도, Netlify 기본 주소(xxx.netlify.app)만으로 사용 가능.

역할 정의 Who does what?

“카페24”와 “Netlify”는 서로 겹치는 부분도 있지만, 기본 역할은 아래처럼 나눌 수 있습니다.

카페24 = 주소(도메인) Netlify = 집(호스팅)
카페24
도메인 & DNS 관리

사람이 기억하기 좋은 이름을 관리합니다.
예: myriskdashboard.com

DNS(A레코드, CNAME)를 통해
“이 주소 → Netlify 서버”로 길 안내를 담당합니다.

Netlify
정적 사이트 호스팅

HTML/CSS/JS 파일이 올라가는 곳입니다.
예: index.html, 그래프 JS 등

프로젝트 폴더를 드래그·드롭 하면
바로 xxx.netlify.app 주소를 발급해 줍니다.

카페24 vs Netlify 비교 도메인(주소)와 호스팅(집)을 나눠 보면 헷갈림이 줄어듭니다.
구분 카페24 (도메인) Netlify (호스팅)
주요 역할 도메인(웹 주소) 등록 · 관리, DNS 설정 정적 웹사이트 파일 저장 · 서비스
대표 URL 형태 my-site.com, bank-risk.co.kr 등 myproject.netlify.app
누가 보나? 사용자가 실제로 입력하는 주소 브라우저가 실제로 접속하는 서버 위치
없어도 되는가? 연습/내부용이면 없어도 됨
(Netlify 기본 주소만 사용 가능) 		웹사이트를 인터넷에 공개하려면
호스팅은 반드시 필요
언제 쓰면 좋은가? 외부 발표, 보고서, 명함 등에서
브랜드 있는 도메인이 필요할 때 		정적 페이지, 리포트, 대시보드를
빠르게 무료로 배포하고 싶을 때
? “처음 연습할 때는 Netlify만, 나중에 진짜 서비스처럼 보이고 싶을 때 도메인을 추가한다”라고 이해하면 편합니다.

이전에 했던 작업 흐름 (카페24 → Netlify)

실제로는 아래처럼 3단계로 진행된 셈입니다.

STEP 1 · 카페24
도메인 구입 또는 설정

myriskdashboard.com 처럼 외우기 좋은 이름을 정하고,
도메인 소유권을 확보합니다.

STEP 2 · Netlify
웹사이트 배포

index.html 등 파일이 들어 있는 폴더를 Netlify에 드래그·드롭해서,
xxx.netlify.app 주소를 하나 받습니다.

STEP 3 · DNS 연결
도메인 → Netlify로 연결

카페24 DNS(A레코드, CNAME)에 Netlify에서 알려준 값을 넣어,
“myriskdashboard.com → 해당 Netlify 사이트”로 길을 안내합니다.

상황별 추천 조합

CASE A 연습 · 내부 공유용
“그냥 빨리 하나 띄우고 싶을 때”
  • 팀 내부에서만 보는 리포트 · 대시보드
  • 개인 공부용 프로젝트, 샘플 화면
  • 주소를 예쁘게 쓸 필요는 없고, 기능만 보이면 될 때

➜ Netlify만 사용해도 충분합니다.
xxx.netlify.app 주소로 공유하면 끝입니다.

CASE B 외부 발표 · 브랜드용
“진짜 서비스처럼 보이고 싶을 때”
  • 외부 발표, 보고서, 명함에 넣을 공식 주소가 필요
  • “이 사이트는 우리 팀/회사 공식 서비스다”라는 느낌을 주고 싶을 때
  • 도메인 이름 자체에 의미를 담고 싶을 때 (예: risk-ailert.com)

➜ 카페24(도메인) + Netlify(호스팅) 조합이 좋습니다.
예쁜 주소 + 유연한 정적 호스팅을 동시에 얻을 수 있습니다.

가장 중요한 포인트는 “주소(도메인)와 집(호스팅)을 분리해서 생각하는 것”입니다.
이 관점만 잡히면, 다른 서비스(가비아+GitHub Pages, AWS Route53+S3 등)를 쓰더라도 개념은 완전히 같게 적용할 수 있습니다.

※ 도메인을 바꾼다고 해서 사이트 내용이 자동으로 옮겨지는 것이 아닙니다.
“내용(파일)”은 호스팅에, “이름(도메인)”은 DNS에 따로 관리된다는 점을 꼭 기억해 주세요.

이 페이지 자체도 index.html로 저장해서 Netlify에 올리면, 동료 교육용 가이드 사이트로 바로 사용할 수 있습니다.
