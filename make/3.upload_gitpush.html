Git & Anaconda · 파이썬 프로젝트 Git Push 5단계
Bird's-eye View

큰 그림은 아래 표처럼 보면 됩니다. 이후 카드에서 각 단계를 세분화해서 설명합니다.

단계 무엇을 하는 단계인가? 핵심 포인트
1단계 Git 호스팅 서비스 가입 (예: GitHub) 계정 생성, 이메일 인증, 새 저장소(Repository) 만들기
2단계 PC에 Git 설치 Git 클라이언트 설치, Git Bash/터미널 사용, 버전 확인
3단계 Anaconda 설치 및 파이썬 환경 구성 파이썬/패키지 관리, Anaconda Prompt에서 Git도 같이 사용
4단계 로컬 프로젝트를 Git 저장소로 초기화 git init, .gitignore, 첫 커밋까지 만들기
5단계 원격 저장소 연결 & git push origin 연결, main 브랜치 설정, git push -u origin main
TIP · 어디에서 명령을 치면 되나?

Git 명령은 보통 Git Bash 또는 터미널(Anaconda Prompt, PowerShell)에서 입력합니다. Anaconda를 설치하면 파이썬 환경과 Git을 같이 쓸 수 있어, “파이썬 프로젝트 폴더에서 곧바로 git push” 흐름이 자연스럽게 이어집니다.

1단계 · Git 호스팅 서비스 가입 (GitHub 기준)

1-1. GitHub 계정 만들기
Account Setup

대표적으로 GitHub 기준으로 설명하지만, GitLab/Bitbucket도 흐름은 거의 같습니다.

  • 브라우저에서 github.com 접속
  • Sign up / 회원가입 버튼 클릭
  • 이메일·비밀번호·사용자명 입력 후 가입 진행
  • 이메일로 전송된 링크를 눌러 이메일 인증
  • 기본 설정은 모두 “다음/Skip”으로 넘겨도 무방
개인 계정 1개로 대부분 충분 업무용이면 회사 메일 사용 고려
1-2. 새 Repository 만들기
Remote Repo

로컬 프로젝트를 올려둘 빈 저장소를 하나 먼저 만들어 둡니다.

  • GitHub 우측 상단 ＋ → New repository 선택
  • Repository name 입력 (예: risk-dashboard)
  • Public / Private 선택
    • 업무/내부용: 보통 Private
  • Initialize this repository with: 부분은 빈 저장소로 두는 것을 추천
    • (로컬에서 이미 프로젝트가 있는 경우, README 생성 체크 해제)
  • Create repository 버튼 클릭
CHECK · 원격 주소 기억해 두기

저장소가 생성되면 상단에 HTTPS / SSH URL이 보입니다. 이후 git remote add origin <URL>에서 사용할 주소입니다.

2단계 · PC에 Git 설치

2-1. Git 다운로드 & 설치
Git Client
  • git-scm.com 접속 후 Download for Windows 클릭
  • 설치 파일 실행 → 대부분 기본 설정으로 “Next” 계속 진행
  • 설치 완료 후 시작 메뉴에서 Git Bash 또는 PowerShell 실행
  • 아래 명령으로 설치 여부 확인: 
    git --version
Git Bash = Git 전용 터미널 Anaconda Prompt에서도 사용 가능
2-2. Git 사용자 정보(Global 설정)
Config

한 번만 설정해 두면, 이후 커밋에 동일하게 기록됩니다.

  • 터미널에서 아래 명령 실행: 
    git config --global user.name "홍길동"

git config --global user.email "sample@example.com"
  • 설정 값 확인: 
    git config --global --list
  • GitHub 계정에 등록한 이메일과 맞추는 것을 추천
TIP

회사/개인 계정을 분리하고 싶다면, --global 대신 프로젝트별로 git config user.name 을 사용해 로컬 설정을 따로 두는 방법도 있습니다.

3단계 · Anaconda 설치 & 파이썬 환경 준비

3-1. Anaconda 다운로드 & 설치
Python Env
  • anaconda.com 접속 → Download 메뉴에서 Windows용 설치 파일 다운로드
  • 설치 실행 후
    • 설치 경로 선택 (기본값 사용 권장)
    • "Add Anaconda to my PATH"는 보통 비활성 (권장 설정대로 진행)
  • 설치 완료 후 시작 메뉴에서 Anaconda Prompt 실행
  • 파이썬 버전 확인: 
    python --version
가상환경 관리에 유리 데이터분석 패키지 기본 포함
3-2. 프로젝트별 가상환경 생성 (선택)
Conda Env

프로젝트마다 별도 환경을 두면 라이브러리 버전 충돌을 줄일 수 있습니다.

  • Anaconda Prompt에서: 
    conda create -n riskproj python=3.11

conda activate riskproj
  • 이제 이 프롬프트에서 프로젝트 폴더로 이동해 작업: 
    cd C:\Users\사용자명\Desktop\my_project
  • 이 터미널에서 git 명령도 그대로 사용 가능
    • Git이 PATH에 잡혀 있으면, 파이썬과 Git을 동시에 사용할 수 있음
TIP

나중에 “requirements.txt” 또는 “environment.yml” 파일을 만들어 두면, 다른 PC에서도 같은 환경을 쉽게 재현할 수 있습니다.

4단계 · 로컬 프로젝트 폴더를 Git 저장소로 만들기

4-1. 파이썬 프로젝트 폴더 준비
Project Folder
  • 예시 경로: 
    C:\Users\사용자명\Desktop\risk_project
  • 폴더 안에는 다음과 같은 구조를 가정:
    • main.py 또는 app.py
    • requirements.txt (또는 환경 파일)
    • data/, notebooks/ 등 서브 폴더
  • 터미널(Anaconda Prompt 또는 Git Bash)에서 해당 폴더로 이동: 
    cd C:\Users\사용자명\Desktop\risk_project
4-2. .gitignore 작성 & git init
Init & Ignore

Git에 올리지 않을 파일/폴더를 먼저 정리한 뒤, 저장소를 초기화합니다.

  • 프로젝트 루트에 .gitignore 파일 생성 (예시): 
    __pycache__/

.ipynb_checkpoints/
.env
*.pyc
.DS_Store
.vscode/
.env/
*.sqlite
*.db
  • 터미널에서 Git 저장소 초기화: 
    git init
  • 현재 상태 확인: 
    git status
NOTE

가상환경 폴더(예: .venv/, env/, conda env 폴더)는 용량이 크므로 반드시 .gitignore에 포함하는 것이 좋습니다.

4-3. 파일 추가 & 첫 커밋 만들기
First Commit
  • 새로 생성/수정된 파일들을 스테이징: 
    git add .
  • 커밋 메시지와 함께 첫 버전을 저장: 
    git commit -m "첫 커밋: 프로젝트 구조 초기 세팅"
  • git status로 “nothing to commit” 상태인지 확인
항상 git status로 상태 확인 작은 단위로 자주 커밋

5단계 · 원격 저장소 연결 & Git Push

5-1. GitHub 저장소를 origin으로 연결
Remote Origin

1단계에서 만든 GitHub 저장소의 URL을 사용합니다.

  • GitHub 페이지에서 HTTPS URL 복사 (예시): 
    https://github.com/username/risk-project.git
  • 터미널에서 origin 원격 저장소 등록: 
    git remote add origin https://github.com/username/risk-project.git
  • 현재 브랜치를 main으로 설정 (이미 main이면 생략 가능): 
    git branch -M main
NOTE

원격 이름은 보통 origin을 사용합니다. 다른 저장소를 추가할 때는 upstream 등 별도 이름을 쓸 수 있습니다.

5-2. 첫 git push 실행
git push -u
  • 첫 push 시: 
    git push -u origin main
  • -u 옵션은 “이 브랜치의 기본 push/pull 대상은 origin/main”이라는 의미로, 다음부터는 단순히 git push만 쳐도 되도록 해 줍니다.
  • HTTPS 기준으로는 처음에 GitHub 로그인/토큰 입력을 요구할 수 있습니다.
  • 정상 완료되면 GitHub 웹 화면에서 코드가 올라간 것을 확인할 수 있습니다.
첫 push만 -u 옵션 이후엔 git push로 간단히
5-3. 이후 파이썬 작업 & Git Push 반복 흐름
Daily Workflow

다음 표처럼 “코드 수정 → 상태 확인 → add → commit → push” 패턴을 반복하면 됩니다.

순서 동작 대표 명령
① 파이썬 코드/노트북 수정 VSCode, Jupyter, PyCharm 등에서 작업
② 변경 사항 확인 git status
③ 스테이징(add) git add 파일명.py 또는 git add .
④ 커밋 메시지와 함께 저장 git commit -m "기능 A 추가, 버그 B 수정"
⑤ 원격 저장소로 업로드 git push
TIP · 파이썬 관점에서의 Git 사용

팀장님이 이미 쓰시는 파이썬/아나콘다 환경 위에, 지금 설명한 Git 워크플로우를 그대로 얹어서 사용하면 됩니다. 즉, 같은 프로젝트 폴더에서 “파이썬 실행”과 “git push”를 같이 한다라고 이해하시면 됩니다.
